Q4. 
        Print output: 
        var x = 21;
        var girl = function () {
            console.log(x);
            var x = 20;
        };
        girl ();

Answer:

Output : "Undefined"

Reason - First var x is declared and initialized globally. so, globally x has a value of 21. But, Second var x is declared inside the "First Class Function" named as girl() so, Hoisting occurs inside girl() function, var x hoists to the top of the function.so, var x is declared again in the functional scope, but not initialized. Hence, the result is Undefined

Q7. Write a function that accepts parameter n and returns factorial of n

Answer:

            function facto(n) {
                var fact = 1;
                for (var i = 1; i <= n; i++) {
                    fact = fact * i;
                }
                return fact;
            }
            console.log(facto(5));

Output : 120

Reason - Here, function facto() is passed with a value 5 as an argument. Inside function, a variable fact is declared and initialized with 1. According to the for loop, var i runs from 1 to n value. Each and everytime i is multiplied with fact which is already present as i value increments. var fact also gets updated for each iteration.

Q9. 
        Guess the output:

        Abc();
        const Abc = function(){
            let value = 20;
            console.log(value);
        }

Answer:

Output : Reference Error!!

Reason - Simply, In javacsript, function declaration and variable declaration are hoisted top of the code, which means only variable is declared and no value is initialized to it. Since, const Abc is a First Class function means the const variable Abc stores the function and the const variable Abc is hoisted to the top and not initialized with the function Abc(). so, we will get a Reference Error.

Q10. 
        Guess the output:

        var a = 10;
        (function (){
            console.log(a);
            var a = 20;
        })();

Answer:

Output : Undefined

Reason - The function is IIFE(Immediately Invoked Function Expression), the function runs as soon as it is defined which is executed using (); Anyhow, the variable var a is declared and initialized in the first line. Then, inside IIFE function again var a is declared and hoisted to top of that function so, var a is declared but not initialized. So, the output is Undefined

Q11.

        const greet =  function(name){
            return function(m){
                console.log(`Hi!! ${name}, ${m}`);
            }

        const greet_message = greet('EA19');
        greet_message("Welcome To PrepBytes");

Answer:

Output : Hi!! EA19, Welcome To PrepBytes

Reason - Firstly, greet("EA19") is invoked as this is a first class function stored into const greet_message. greet function returns another function(m) with parameter m. greet_message(m) is then invoked with the message passed through its argument and in console it prints the both the values passed in the arguments.
